# Security Policy

## Reporting A Vulnerability

Do not report sensitive security concerns in public issues.

Use GitHub private vulnerability reporting when available, or contact the
maintainers through the private channel listed in the repository profile.

Include:

- a clear description of the concern;
- public reproduction steps when possible;
- impact and affected case files or scripts;
- whether any secret, non-public material, or maintainer correspondence
  may have been exposed.

## Sensitive Data Boundary

If a secret or private artifact is accidentally published, revoke or rotate the
secret first, then coordinate repository cleanup. Public deletion alone is not a
complete remediation because repository history and forks may retain content.

## Supported Surface

This repository contains public documentation and case records. It does not
publish SDS product materials or target repository clones.
